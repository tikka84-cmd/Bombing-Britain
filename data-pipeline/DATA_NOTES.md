# DATA_NOTES

Placeholder. This file is generated by `01_inspect` and currently describes
nothing because the raw dataset has not been downloaded yet.

## How to populate it

1. Download the "Bombing Britain: an air raid map" dataset (HO 203 transcription,
   ~32,000 attacks). The project page is
   `http://www.warstateandsociety.com/Bombing-Britain`, which now redirects to
   History Commons. The site blocks automated downloads, so this must be done by
   hand in a browser.
2. Save the file into `data-pipeline/raw/` (it stays gitignored while the reuse
   licence is unresolved, see the project brief, section 3).
3. From `data-pipeline/`, run:

   ```
   npm run inspect
   ```

   or, if the file is not auto-detected:

   ```
   node scripts/01_inspect.mjs raw/<filename>
   ```

The script reads CSV / TSV / semicolon-delimited text. If the download is an
Excel file, open it and "Save As" CSV first.

It will overwrite this file with the real summary: column names, row count, the
full list of distinct attack-type strings, how casualty cells encode
"Unspecified" and vague descriptors, Additional Notes patterns, per-year row
counts, and a sample of late-war (1944-1945) rows for the V-2 under-recording
check.
